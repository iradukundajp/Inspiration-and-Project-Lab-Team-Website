Introduction

The Neighborhood Resource Sharing App is a community-based platform designed to help local residents lend, borrow, and request everyday items from one another. Instead of buying tools, furniture, or household items that are only needed occasionally, users can easily find and borrow them from nearby neighbors—building trust, saving money, and reducing waste.

Why This App Matters

Example 1

You need a drill for 5 minutes — don't buy it. Borrow it.

Example 2

You bought a party table but only use it once a year. Let others borrow it!

Example 3

You moved into a new place and need cleaning tools — get them from your neighbor.

This app helps people live smarter, waste less, and feel more connected with their community.

Project Objectives

  • Facilitate Item Sharing: Allow neighbors to list and find items available to borrow.
  • Reduce Waste: Encourage reuse rather than constant buying.
  • Build Local Trust: Promote a helpful and engaged local community.
  • Provide a Clean Interface: Design a user-friendly platform that works well on all devices.

Core Features

User Management

  • Registration & Login: Users can create accounts with name, email, password, and location.
  • Profile Page: Users view/edit their info and items they're sharing.
  • Admin Role: Admins can moderate users, items, and reviews.

Item Management

  • Add Item: Users can add items with title, category, image, and availability.
  • Search/Browse: Filter by category, location, availability.
  • Item Details: View description, photo, owner info, and borrow options.

Borrowing System

  • Request to Borrow: Send custom messages and date ranges.
  • Borrow History: View past lends and borrows.

Messaging System

  • Chat: Coordinate pickups and ask questions.

Reviews & Ratings

  • Leave Reviews: Rate and review transactions.
  • Trust System: Build credibility through feedback.

Database Relationships

  • Users (1) → (Many) Items via OwnerID
  • Users (1) → (Many) BorrowRequests via RequesterID
  • Items (1) → (Many) BorrowRequests via ItemID
  • Users (1) → (Many) Messages as Sender via SenderID
  • Users (1) → (Many) Messages as Receiver via ReceiverID
  • Users (1) → (Many) Reviews as FromUser via FromUserID
  • Users (1) → (Many) Reviews as ToUser via ToUserID
  • BorrowRequests (1) → (Many) Reviews via RequestID
  • Users (1) → (Many) SystemLogs via UserID

ERD (Entity Relationship Diagram)

A comprehensive diagram showing connections between Users, Items, BorrowRequests, Messages, Reviews, and SystemLogs.

Key Relationships:

  •  Users can own multiple Items and make multiple BorrowRequests
  •  Items can have multiple BorrowRequests from different users
  •  Messages connect users for communication about items and requests
  •  Reviews are linked to completed transactions between users
  •  SystemLogs track all user activities for monitoring and debugging

UML (Unified Modeling Language Diagram)

A UML class diagram illustrating the relationships, methods, and properties of core system classes.

Core Classes Include:

  • User Class: Manages user credentials, profile, and authentication
  • Item Class: Handles resource details, availability, and categories
  • BorrowRequest Class: Manages borrowing transactions and status
  • Message Class: Facilitates communication between users
  • Review Class: Handles ratings and feedback system
  • SystemLog Class: Tracks all user activities for monitoring and debugging
  • UserManager Class: Handles user registration, authentication, and profile management
  • DataManager Class: Manages database connections and data validation
  • DBConnection Class: Handles database connectivity and connection pooling
  • AppUtilities Class: Provides common utility functions and validation methods
